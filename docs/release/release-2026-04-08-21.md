## Risks And Rollback

- The split is intended to be behavior-preserving; the main risk is future drift if shared upstream request/url construction is copied back into `local_ai_proxy.rs` or reimplemented independently by probe/request-serving code.
- Shared upstream helpers now live behind one explicit submodule owner; later refactors must preserve that boundary rather than fragment it again.
- Rollback is limited to the listed Rust/script files and the associated review, architecture, and release writebacks.
