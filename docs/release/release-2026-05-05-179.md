- Publishes the post-173 kernel authority, chat workspace, shared SDK Git-source, and Windows packaged-launch smoke hardening with local relative SDK development dependencies preserved.
- Keeps release-mode shared SDK dependencies resolved from pinned GitHub repositories instead of npm-published central packages.
- Authenticates private GitHub shared SDK clones in CI and release workflows with `SDKWORK_SHARED_SDK_GITHUB_TOKEN`, falling back to `github.token` when a dedicated secret is not configured.
- Keeps configured GitHub repository URLs visible in release logs while passing credentials through Git `extraheader`; tokens are not embedded into clone URLs.
- Redacts GitHub `extraheader` values from shared SDK source helper error messages so failed clone, fetch, or `ls-remote` diagnostics do not leak token material.
- Pins CI and release Rust builds to the locally verified `1.90.0` toolchain so release output is not affected by future floating `stable` compiler regressions.
- Keeps Windows Tauri release bundling on `CARGO_PROFILE_RELEASE_STRIP=none` and `CARGO_PROFILE_RELEASE_OPT_LEVEL=2`, and moves the release toolchain off the locally reproduced `rustc 1.91.1` plus `windows-sys 0.61.2` const-eval failure path.
- Syncs `pnpm-lock.yaml` with `packages/sdkwork-claw-ui/package.json` so CI and release runners can install with `pnpm install --frozen-lockfile` after `@sdkwork/claw-ui` added `qrcode@^1.5.4`.
- Adds POSIX `sdkwork-run-node` and `sdkwork-run-pnpm` workspace wrappers and exposes the checked-out workspace root through `GITHUB_PATH` in CI and release jobs, so Linux and macOS runners can execute the same package lifecycle scripts that Windows resolves through `.cmd` wrappers.
- Keeps the wrapper PATH setup compatible with Windows Git Bash by resolving chmod paths through `cygpath` while preserving the native `GITHUB_PATH` entry GitHub Actions expects.
- Exposes `SDKWORK_GENERATOR_ROOT` to shared SDK package build subprocesses, allowing clean-room GitHub release jobs to build the relocated IM/RTC SDK repositories through the installed claw-studio workspace toolchain when the broader `sdk/sdkwork-sdk-generator` checkout is not present.
- Updates the `@sdkwork/im-sdk` release pin to `12abdbd638d499194e89cf966ed0aa4cd103f19b`, which adds `compilerOptions.rootDir = "./src"` to the TypeScript single-package build config and wires a root-tsconfig contract test into the IM SDK verification entrypoint.
