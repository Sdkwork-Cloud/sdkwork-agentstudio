# Agent Studio release-2026-05-05-175

## Highlights

- Carries forward the complete `release-2026-05-05-174` payload because tag `release-2026-05-05-174` did not publish a GitHub Release.
- Publishes the post-173 kernel authority, chat workspace, shared SDK Git-source, and Windows packaged-launch smoke hardening with local relative SDK development dependencies preserved.
- Keeps release-mode shared SDK dependencies resolved from pinned GitHub repositories instead of npm-published central packages.
- Authenticates private GitHub shared SDK clones in CI and release workflows with `SDKWORK_SHARED_SDK_GITHUB_TOKEN`, falling back to `github.token` when a dedicated secret is not configured.
- Keeps configured GitHub repository URLs visible in release logs while passing credentials through Git `extraheader`; tokens are not embedded into clone URLs.
- Redacts GitHub `extraheader` values from shared SDK source helper error messages so failed clone, fetch, or `ls-remote` diagnostics do not leak token material.
- Pins CI and release Rust builds to the locally verified `1.91.1` toolchain so release output is not affected by future floating `stable` compiler regressions.
- Stabilizes Windows Tauri release bundling under Rust `1.91.1` by setting `CARGO_PROFILE_RELEASE_STRIP=none` and `CARGO_PROFILE_RELEASE_OPT_LEVEL=2`, matching the local Windows release build profile that avoided reproduced const-eval and `STATUS_ACCESS_VIOLATION` failures in the full desktop dependency graph.
- Preserves the Windows packaged desktop launch smoke isolation from `release-2026-05-05-174` by honoring explicit `SDKWORK_CLAW_USER_ROOT` and `SDKWORK_CLAW_MACHINE_ROOT` roots inside the Tauri path resolver.
