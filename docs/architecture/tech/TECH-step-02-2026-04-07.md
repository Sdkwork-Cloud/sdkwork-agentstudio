# Step 02 导出与依赖方向决议 - 2026-04-07

## 1. 决议

1. `sync-feature-packages` 的同步范围限定为真实 feature 包；`host-core`、`host-studio`、`server` 明确排除。
2. feature 包若没有 `components/` 目录，不得因同步脚本失败，也不得被强行创建空目录。
3. package root 导出以顶层 wrapper 与显式顶层 barrel 为准，不再把 `pages/...` 深层实现路径写入公共面。
4. `check:sdkwork-structure` 必须附带 `sync-feature-packages` 回归测试，防止 feature wiring 漂移。
5. `check:arch` 与 `check:sdkwork-hosts` 同时守护顶层结构，避免 Host/Shell 回退为“能跑但不受控”。

## 2. 影响面

- 多个 feature 包的 `package.json` 依赖已按实际 import 重算。
- 多个 feature 包的 `src/index.ts`、`src/services/index.ts`、`src/components/index.ts` 已按同步规则收口。
- Host/Shell 边界已从“约定”升级为“脚本门禁”。

## 3. 下一步约束

- `Step 03` 不得再把 Runtime/OpenClaw/Proxy 逻辑塞回 Host 根目录。
- `Step 04` 的 API 契约改造必须继续沿用 package root 公共面，不允许反向引入内部路径依赖。
