# Claw Studio release-2026-04-10-164

## Highlights

- Preserved installed-skill instance-asset metadata through the `mySkillService` readback path instead of dropping workbench state at the Market boundary.
- Added direct helper coverage plus contract enforcement for installed-skill `source` / `scope` / `status` / `compatibility` / missing-requirement readback.
- Wrote the first Step 08 review, architecture, and release evidence for the installed-skill readback lane while keeping `CP08-3` explicitly open.

## Attempt Outcome

- Production-code scope in this Step 08 loop:
  - `packages/sdkwork-clawstudio-types/src/index.ts`
    - added optional `Skill.instanceAsset`
    - added explicit instance-asset scope/status/compatibility metadata types
  - `packages/sdkwork-clawstudio-market/src/services/mySkillService.ts`
    - maps installed-skill workbench metadata into `Skill.instanceAsset`
    - derives `status` as `enabled | disabled | blocked`
    - derives `compatibility` as `compatible | attention | blocked`
    - preserves optional `filePath` / `baseDir`
    - keeps uninstall authority on `agentSkillManagementService.removeSkill` / `setSkillEnabled`
  - `packages/sdkwork-clawstudio-market/src/services/mySkillService.test.ts`
    - covers disabled and blocked installed-skill readback cases
  - `scripts/sdkwork-market-contract.test.ts`
    - now requires `instanceAsset`, `compatibility`, and `missingRequirementCount` preservation
- Writeback scope in this loop:
  - `docs/review/step-08-installed-skill-instance-asset-readback-2026-04-10.md`
  - `docs/架构/08-ClawHub、Channels与生态扩展设计.md`

- Real runtime truth for this loop remains:
  - install path: `marketService -> agentSkillManagementService.installSkill -> gateway.skills.install`
  - readback path: `mySkillService -> agentWorkbenchService.getAgentWorkbench() -> gateway.skills.status` projection
  - uninstall path: `agentSkillManagementService.removeSkill` or `setSkillEnabled`
- Step boundary for this loop:
  - catalog-side `SkillDetail` / `SkillPackDetail` compatibility copy is still static
  - therefore this loop advances `CP08-3` but does not close `Step 08`

## Risks And Rollback

- `Skill.instanceAsset` is intentionally optional, so catalog skills and packs remain backward-compatible in this loop.
- The main product risk is semantic drift if a later Step 08 loop starts fabricating catalog compatibility or provenance without routing it from a real runtime truth source.
- Rollback is limited to:
  - the installed-skill instance-asset typing and Market readback mapping
  - the focused test/contract guardrails
  - this loop's review/architecture/release records
